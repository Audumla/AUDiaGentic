# Description

Implement `tm_archive` and `tm_restore` functions for managing archived planning objects.

## tm_archive Function

**Signature:**
```
tm_archive(id: string, reason: string, archived_by: string) -> archived_object
```

**Parameters:**
- `id`: Object ID to archive (required)
- `reason`: Why the object is being archived (required)
- `archived_by`: User/system performing the archive (required)

**Validation:**
1. Object ID exists
2. Object is in `done` state
3. All required parameters provided

**Behavior:**
1. Validate object exists and is in `done` state
2. Set state to `archived`
3. Add archive_metadata:
   ```yaml
   archive_metadata:
     is_archived: true
     archived_at: <current_timestamp>
     archived_by: <archived_by>
     archive_reason: <reason>
   ```
4. Preserve all relationships and metadata
5. Return updated object

**Error Responses:**
- `400`: Object not in `done` state
- `404`: Object ID not found
- `400`: Missing required parameters

## tm_restore Function

**Signature:**
```
tm_restore(id: string, restored_to_state: string, restored_by: string) -> restored_object
```

**Parameters:**
- `id`: Archived object ID (required)
- `restored_to_state`: State to restore to (draft/ready/in_progress) (required)
- `restored_by`: User/system performing restoration (required)

**Validation:**
1. Object ID exists
2. Object is in `archived` state
3. restored_to_state is valid (draft/ready/in_progress)
4. All required parameters provided

**Behavior:**
1. Validate object exists and is in `archived` state
2. Validate restored_to_state is valid
3. Remove archive_metadata.is_archived
4. Set state to restored_to_state
5. Add restoration metadata:
   ```yaml
   archive_metadata:
     restored_at: <current_timestamp>
     restored_to_state: <restored_to_state>
     restored_by: <restored_by>
   ```
6. Return updated object

**Error Responses:**
- `400`: Object not in `archived` state
- `400`: Invalid restored_to_state value
- `404`: Object ID not found
- `400`: Missing required parameters

# Acceptance Criteria

1. `tm_archive` archives objects in `done` state
2. `tm_archive` rejects objects not in `done` state
3. `tm_archive` includes archive_metadata in response
4. `tm_restore` restores archived objects
5. `tm_restore` rejects non-archived objects
6. `tm_restore` updates state correctly
7. Archive metadata preserved through operations
8. Error messages clear and actionable

# Notes

- Archive metadata should be optional field in data model
- Timestamps should use ISO 8601 format
- Consider adding soft delete option for future enhancement
