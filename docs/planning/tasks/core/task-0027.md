# Description

Extend the MCP planning tools to allow agents to create and update full file content, not just metadata.

## Current Limitations

**tm_update only supports:**
- Updating metadata fields (label, summary)
- Appending text to end of file

**What's missing:**
- ❌ Replace entire content
- ❌ Update specific sections
- ❌ Insert content at specific positions
- ❌ Format structured content (tables, code blocks)

## Required Functions

### tm_update_content

**Signature:**
```
tm_update_content(id: string, content: string, mode: string) -> updated_object
```

**Parameters:**
- `id`: Object ID to update
- `content`: Full markdown content (without YAML frontmatter)
- `mode`: Update mode (replace/append/insert/section)

**Modes:**

1. **replace** (default): Replace entire content
   ```python
   tm_update_content(id="spec-0001", content="# New content\n\nMore text", mode="replace")
   ```

2. **append**: Append to end (current behavior)
   ```python
   tm_update_content(id="spec-0001", content="\n\n# New section", mode="append")
   ```

3. **insert**: Insert at specific position
   ```python
   tm_update_content(id="spec-0001", content="# New section", mode="insert", position=2)
   ```

4. **section**: Update specific section by header
   ```python
   tm_update_content(id="spec-0001", content="# Requirements\n\nNew requirements", mode="section", section="# Requirements")
   ```

### tm_create_with_content

**Signature:**
```
tm_create_with_content(kind: string, label: string, summary: string, content: string, **refs) -> created_object
```

**Parameters:**
- `kind`: Object type (plan/request/spec/task/wp)
- `label`: Object label
- `summary`: Object summary
- `content`: Full markdown content
- `**refs`: Reference parameters (spec_ref, request_refs, etc.)

**Example:**
```python
tm_create_with_content(
    kind="spec",
    label="Migration spec",
    summary="Migrate docs to planning structure",
    content="# Purpose\n\nMigrate all docs\n\n# Scope\n\nAll implementation docs",
    request_refs=["request-0001"]
)
```

### tm_get_content

**Signature:**
```
tm_get_content(id: string) -> content
```

**Returns:**
- Full markdown content (without YAML frontmatter)

**Example:**
```python
content = tm_get_content("spec-0001")
# Returns: "# Purpose\n\nMigrate all docs..."
```

## YAML Frontmatter Handling

**Automatic handling:**
- Frontmatter preserved during content updates
- Frontmatter extracted before content operations
- Frontmatter re-added after content operations

**Example:**
```python
# File content:
---
id: spec-0001
label: Old label
---
# Content here

# Operation:
tm_update_content(id="spec-0001", content="# New content", mode="replace")

# Result:
---
id: spec-0001
label: Old label
---
# New content
```

## Section-Based Updates

**Section detection:**
- Parse markdown headers (#, ##, ###)
- Identify section boundaries
- Replace entire section including header

**Section update algorithm:**
1. Find section header in content
2. Extract section start (header line)
3. Find next section or end of file
4. Replace section with new content

**Example:**
```python
# Before:
# Purpose
# Old purpose text

# Requirements
# Old requirements

# After tm_update_content with mode="section", section="# Requirements":
# Purpose
# Old purpose text

# Requirements
# New requirements text
```
