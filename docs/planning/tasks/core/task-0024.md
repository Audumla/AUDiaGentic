# Description

Update query functions to support filtering archived objects.

## tm_list Update

**Current Signature:**
```
tm_list(kind: string)
```

**Updated Signature:**
```
tm_list(kind: string, include_archived: boolean)
```

**Parameters:**
- `kind`: Object type (plan/request/spec/task/wp)
- `include_archived`: Include archived objects (default: false)

**Behavior:**
- When `include_archived=false` (default): Only returns active objects
- When `include_archived=true`: Returns all objects including archived

**Implementation:**
```python
def tm_list(kind: str, include_archived: bool = False):
    objects = get_objects_by_kind(kind)
    if not include_archived:
        objects = [obj for obj in objects if not obj.archive_metadata.is_archived]
    return objects
```

## tm_next_tasks Update

**Current Behavior:**
- Returns tasks in `ready` state

**Updated Behavior:**
- Returns tasks in `ready` state
- Excludes archived tasks automatically
- No parameter needed (archive exclusion is implicit)

**Implementation:**
```python
def tm_next_tasks(state: str = "ready", domain: str = None):
    tasks = get_tasks_by_state(state)
    # Always exclude archived tasks
    tasks = [t for t in tasks if not t.archive_metadata.is_archived]
    if domain:
        tasks = [t for t in tasks if t.domain == domain]
    return tasks
```

## tm_next_items Update

**Updated Behavior:**
- Excludes archived objects automatically
- Works for any kind (task, plan, spec, etc.)

**Implementation:**
```python
def tm_next_items(kind: str = "task", state: str = "ready", domain: str = None):
    items = get_next_items_by_kind(kind, state)
    # Always exclude archived items
    items = [i for i in items if not i.archive_metadata.is_archived]
    if domain:
        items = [i for i in items if i.domain == domain]
    return items
```

# Acceptance Criteria

1. `tm_list(kind)` excludes archived objects by default
2. `tm_list(kind, include_archived=true)` includes archived objects
3. `tm_next_tasks()` excludes archived tasks automatically
4. `tm_next_items()` excludes archived objects automatically
5. Performance impact minimal (simple filter)
6. Backward compatible (default behavior unchanged)

# Notes

- Default behavior should not change for existing callers
- Archive filtering should be efficient (indexed field if possible)
- Consider adding archive filter to other query functions as needed
