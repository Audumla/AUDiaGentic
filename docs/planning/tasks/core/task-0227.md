# Description

Two changes are required:

**1. Body content parameters for `tm.py new task`**

Add `--description`, `--acceptance`, and `--notes` optional arguments to the `new task` subcommand in `tools/planning/tm.py`. Pass these through to `task_mgr.create()` which should write them into the corresponding markdown body sections instead of leaving them blank.

The body template in `task_mgr.py` currently generates:

```text
# Description\n\n\n# Acceptance Criteria\n\n\n# Notes\n\n
```

When the parameters are supplied, replace the empty section bodies with the provided content.

## Validation: non-empty body sections

Add checks in `val_mgr.py` that a task's markdown body contains non-empty `Description` and `Acceptance Criteria` sections. A task with blank sections must fail validation with a clear message, blocking state transitions via the hook.

# Acceptance Criteria

- `tm.py new task --label "..." --summary "..." --spec spec-NNNN --description "..." --acceptance "..." --source claude` creates a task with populated body sections in one command
- `--description`, `--acceptance`, and `--notes` are all optional; omitting them produces the current blank stub behaviour
- A task with an empty `Description` or `Acceptance Criteria` section fails validation
- The validation error message identifies the task id and the missing section
- All existing tests pass
- New tests cover: fully-populated creation, blank-section validation failure, and state-transition block on empty body

# Notes

The `--notes` parameter is optional and not required for validation — notes are informational only. The validation check should mirror the existing section-presence checks in `val_mgr.py`.
