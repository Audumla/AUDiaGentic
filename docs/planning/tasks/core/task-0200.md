# Description

Improve schema validation error messages to be actionable and clear. Current errors like `'wp-0010' is not of type 'object'` don't indicate what format is expected.

**Current behavior:**
```
'wp-0010' is not of type 'object'
```

**Required behavior:**
```
work_package_refs must be a list of objects with 'ref' (string) and 'seq' (integer) fields.
Expected: - ref: wp-0010\n  seq: 1000
Got: wp-0010
```

# Acceptance Criteria

1. Validation errors include expected format examples
2. Validation errors indicate which field has the issue
3. Validation errors suggest correct format for reference fields (ref/seq objects)
4. Common high-friction mistakes get clearer messages without replacing the whole validator
5. `tm_validate()` output is human-readable and actionable
