# Description

Scan all implementation documentation files, extract all file references, and build a comprehensive bidirectional reference map to guide the migration.

## Scope

- Scan `docs/implementation/*.md` files
- Scan `docs/implementation/**/*.md` (subdirectories)
- Extract all markdown file references (e.g., `01_Master_Implementation_Roadmap.md`)
- Extract all directory references (e.g., `packets/phase-0/`)
- Build reference map showing:
  - Which files reference which other files
  - Which files are referenced by others
  - Reference types (doc, packet, directory)

## Reference Patterns

### File References
- `[01_Master_Implementation_Roadmap.md](path/to/file.md)`
- `- \`02_Phase_Gates_and_Exit_Criteria.md\``
- `See 03_Target_Codebase_Tree.md`

### Directory References
- `packets/phase-0/`
- `packets/phase-1-1/ through phase-1-3/`

### Cross-Document References
- Internal links within implementation docs
- Links to planning docs
- Links to specifications

# Acceptance Criteria

1. Complete inventory of all `docs/implementation/*.md` files
2. Reference map showing all inter-file dependencies
3. Identification of root files (no incoming references)
4. Identification of leaf files (no outgoing references)
5. Migration order recommendation based on dependencies
6. Output: `docs/planning/tasks/core/migration-reference-map.md`

# Notes

- This task must complete before any migration begins
- Reference map will guide the migration order
- Files with no dependencies can be migrated first
- Files with many dependencies should be migrated later

# Output

Reference map created at: `docs/planning/tasks/core/migration-reference-map.md`

Contains:
- Complete inventory of 234 markdown files
- Reference analysis showing root files and dependencies
- Migration order recommendation (9 phases)
- Cross-reference patterns
- Next steps for migration
