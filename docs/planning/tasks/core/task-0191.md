# Description

Create tm_archive() and tm_restore() helper functions for archiving and restoring planning items.

**Status: IN PROGRESS**

## Requirements

### tm_archive()

```python
def tm_archive(
    id_: str,
    reason: str | None = None,
    root: Path | None = None,
) -> dict[str, Any]:
    """Archive a planning item.
    
    Args:
        id_: Item ID to archive
        reason: Optional reason for archiving
        root: Optional project root
    
    Returns:
        Dict with id, state, archived_at, archived_by, reason
    """
```

### tm_restore()

```python
def tm_restore(
    id_: str,
    root: Path | None = None,
) -> dict[str, Any]:
    """Restore an archived planning item.
    
    Args:
        id_: Item ID to restore
        root: Optional project root
    
    Returns:
        Dict with id, state, restored_at, restored_by
    """
```

## Implementation

- Add to `tools/planning/tm_helper.py`
- Call `api.state()` with appropriate state transitions
- Record archive metadata (archived_at, archived_by, reason)
- Log archive events

## Acceptance Criteria

1. tm_archive() transitions item to archived state
2. tm_restore() transitions item back to ready state
3. Archive metadata is recorded (timestamp, user, reason)
4. Archive events are logged
5. Archived items cannot be modified directly
