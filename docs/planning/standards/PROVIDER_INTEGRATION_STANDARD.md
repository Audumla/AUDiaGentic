# Adding a New Provider to AUDiaGentic

When adding support for a new provider (agent, LLM, or tool), follow this checklist to ensure rules, documentation, and generation are properly configured.

## Step 1: Create Provider-Specific Rule Files

Add rule files to `.audiagentic/config/rules/providers/<provider-name>/`:

```
.audiagentic/config/rules/providers/<provider-name>/
├── 01-provider-specific.md          # Provider-specific constraints/capabilities
└── (additional rules as needed)
```

**Example content** (for new provider "example-ai"):
```markdown
## Example-AI Specific Instructions

[Provider-specific behavior, authentication, rate limits, etc.]
```

## Step 2: Determine Output Format(s)

Decide what agent instruction files this provider needs:

| Provider | Output File | Format | Notes |
|----------|------------|--------|-------|
| Claude | CLAUDE.md | Markdown | Plain text, human-readable |
| Codex | CODEX.md | Markdown | Plain text, human-readable |
| Cline | .clinerules | Custom | Cline-specific format |
| Copilot | .github/copilot-instructions.md | Markdown | GitHub location |
| New Provider | ? | ? | Define output file and format |

## Step 3: Add Generator Logic

Update the rule generator (or create one if it doesn't exist) to handle the new provider:

**File:** `tools/config/generate_provider_rules.py` (or equivalent)

Add a function:
```python
def generate_<provider_name>_rules(shared_rules, provider_rules):
    """Generate <PROVIDER-NAME> instruction file from rule set."""
    output = "<!-- Generated by AUDiaGentic rule generator -->\n\n"
    output += _merge_rules(shared_rules, provider_rules)
    return output
```

Register the generator:
```python
PROVIDERS = {
    'claude': (generate_claude_rules, 'CLAUDE.md'),
    'codex': (generate_codex_rules, 'CODEX.md'),
    '<new-provider>': (generate_<provider_name>_rules, '<OUTPUT_FILE>'),
}
```

## Step 4: Update Rule Synchronization

Add the new provider to the list of synchronized files in `CLAUDE.md`:

```markdown
All synced files: `CLAUDE.md` `CODEX.md` `<NEW_FILE>` (run generator to regenerate)
```

## Step 5: Test Provider Integration

1. **Verify rule file creation:**
   ```bash
   python tools/config/generate_provider_rules.py
   ```

2. **Check output file:**
   - File exists at expected path
   - Content includes shared rules
   - Content includes provider-specific rules
   - Formatting is correct for the provider

3. **Verify sync policy:**
   - Provider-specific file is referenced in CLAUDE.md
   - Comments indicate automatic generation
   - Header matches other generated files

4. **Test rule application:**
   - Provider can read and understand the rules
   - Rules are enforced in practice (no contradictions)

## Step 6: Document Provider-Specific Behaviors

Add provider-specific behavior notes to `.audiagentic/config/PROVIDERS.md`:

```markdown
## <Provider Name>

**Output file:** `<OUTPUT_FILE>`  
**Format:** `<FORMAT>`  
**Frequency:** Rules generated via `generate_provider_rules.py`  
**Special considerations:**
- [Any specific behaviors or constraints]
```

## Step 7: Checklist

Before considering the provider integration complete:

- [ ] Provider-specific rule files created in `.audiagentic/config/rules/providers/<provider>/`
- [ ] Output file format determined
- [ ] Generator logic implemented and tested
- [ ] New provider added to PROVIDERS dict
- [ ] Output file regenerated successfully
- [ ] CLAUDE.md updated to list new output file
- [ ] New output file has proper header/comment
- [ ] Provider documentation added to PROVIDERS.md
- [ ] All shared rules properly merged into provider output
- [ ] No rule conflicts between shared and provider-specific rules
- [ ] Tested with actual provider (can it read and apply the rules?)

## Maintenance

When updating shared rules:
1. Edit files in `.audiagentic/config/rules/shared/`
2. Run `python tools/config/generate_provider_rules.py`
3. Commit both the rule changes and generated output files
4. Verify all providers have updated rules
