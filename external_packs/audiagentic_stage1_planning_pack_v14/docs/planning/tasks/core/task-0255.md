# Description

Freeze one generic resolve and reconcile contract shared by install, update, validate, status, and doctor.

# Likely surfaces

- resolver or normalization helpers under `src/audiagentic/runtime/lifecycle/`
- validation helpers under `src/audiagentic/foundation/contracts/`
- backend planning hooks under installer or lifecycle modules

# Must define

- input precedence order
- observed-state inputs
- plan-step derivation rule
- mode differences without semantic drift

# Do not change

- do not create separate resolve logic per command
- do not make apply-only logic the source of truth for doctor or validate

# Done means

- one shared resolution contract can drive all stage-one commands
- desired-state, observed-state, and realized-capability remain distinct
